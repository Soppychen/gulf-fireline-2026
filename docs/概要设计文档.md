# 《海湾火线 2026》概要设计文档

版本：0.1  
目标平台：Web 首发，后续扩展 Android  
Demo 视角：美国空军  
核心类型：纵向卷轴空战射击  

## 1. 项目定位

《海湾火线 2026》是一款致敬《1942/1943/1944》系列的现代空战题材纵向卷轴射击游戏。玩家驾驶现代战机，在沙漠、山谷、城市外围、防空阵地和地下设施上空执行高压突防任务，通过方向键或触摸拖拽控制飞机移动，自动射击敌方单位，躲避敌机、导弹、无人机、防空火力和环境威胁。

游戏使用现实新闻与公开军事装备作为视觉和世界观参考，但整体设定采用架空化处理：不使用真实任务细节、真实人物、真实部队番号、真实坐标和现实可操作战术信息。关卡目标应表达为“摧毁虚构指挥节点”“压制防空网络”“护送侦察平台”等游戏化任务。

## 2. 设计原则

- 街机优先：手感、节奏、可读性优先于军事写实。
- 现实启发，架空表达：机体和建筑可参考公开视觉特征，但所有阵营标识、地图、行动名称和目标都应虚构。
- 不鼓励现实仇恨：多阵营视角用于呈现不同立场和玩法，而不是煽动现实敌意。
- 低门槛，高上限：玩家无需理解复杂军事系统，也能通过移动、躲避、升级获得爽感。
- Web 优先：Demo 以浏览器流畅运行为目标，移动端适配从一开始纳入输入和布局设计。

## 3. 世界观与阵营

游戏设定在架空近未来的海湾危机中。三个可切换阵营拥有不同飞机、任务目标和关卡叙事。

### 3.1 Demo 阵营：美国空军

当前 Demo 只开放美国空军视角。玩家驾驶隐身战斗机执行第一阶段突防任务，目标是在敌方防空系统完全展开前，对虚构的高价值指挥节点进行闪电攻击。

Demo 建议玩家机体：

- AFX-35：参考 F-35/F-22 的隐身战斗机轮廓，均衡型。
- 后续可解锁 B-2 风格隐身轰炸机，但 Demo 第一版不必实现。

### 3.2 后续阵营：以色列方面

以色列方面可偏向高机动、高精度、小编队作战。玩法上可强调低空穿插、电子干扰、短时间爆发输出。

可能机体：

- I-35I：参考 F-35I 的原创变体。
- 沙漠鹰中队机：参考 F-15/F-16 的原创变体。

### 3.3 后续阵营：伊朗方面

伊朗方面可偏向本土防空、无人机反击、导弹拦截与阵地防御。玩法不一定只做“驾驶飞机”，也可以做防空射击、无人机突袭、地面防御关卡，形成阵营差异。

可能玩法：

- 防空炮/导弹阵地防御模式。
- 无人机蜂群突袭模式。
- 老式战机高风险拦截模式。

## 4. Demo 第一关设计

关卡名：闪电攻击  
任务类型：高价值指挥节点突袭  
玩家视角：美国空军  
推荐时长：3-4 分钟  
核心体验：快速进入、强节奏突防、让玩家理解移动、自动射击、拾取强化、躲避导弹和击败 Boss。

### 4.1 叙事包装

任务简报：

> 敌方沿山地走廊部署了临时指挥链路和防空节点。玩家小队需要在防空系统完全联网前，穿越雷达阵地、无人机巡逻线和导弹车队，对虚构的移动指挥中心进行一次快速打击。

注意：

- 不出现现实领导人姓名。
- 不出现真实地点坐标。
- 不呈现现实暗杀或斩首细节。
- 目标命名为“移动指挥节点”“战区通信核心”“防空链路中枢”等虚构军事目标。

### 4.2 场景流程

1. 开场：夜间沙漠和山脉上空，少量无人机巡逻。
2. 第一段：轻型敌机与地面雷达站，教学自动射击和拾取火力升级。
3. 第二段：防空炮和导弹车出现，教学横向躲避和导弹预警。
4. 第三段：无人机蜂群从左右包夹，强调走位。
5. 小 Boss：移动指挥车队，包含护卫车、防空炮和通信车。
6. 结算：根据剩余生命、连击、击毁率和用时给出评分。

### 4.3 第一关单位

玩家：

- AFX-35 隐身战斗机
- 初始生命：3
- 默认武器：直线机炮
- 可获得武器：机载追踪导弹，通过导弹补给道具临时解锁
- 主动技能：短时护盾或 EMP 脉冲，二选一先做

敌方空中单位：

- 巡逻无人机：低血量，直线下行。
- 拦截无人机：朝玩家当前位置斜向俯冲。
- 轻型战机：横向入场，发射少量子弹。

敌方地面单位：

- 雷达站：存在时提升导弹锁定频率。
- 防空炮：固定扇形弹幕。
- 导弹车：周期性发射低速地对空追踪导弹。
- 固定式 SAM 发射架：中后段出现，发射前有明显锁定预警和发射烟迹。

小 Boss：

- 移动指挥车队：由 1 个通信车、2 个防空车、若干护卫车组成。
- 阶段 1：护卫车发射普通弹。
- 阶段 2：防空车发射扇形弹幕。
- 阶段 3：通信车开启护盾并召唤无人机。

## 5. 核心玩法系统

### 5.1 移动与射击

- 玩家通过方向键、WASD 或触摸拖拽控制飞机。
- 玩家飞机自动射击。
- 桌面端支持 Shift 低速精密移动。
- 移动端优先使用“手指拖拽飞机跟随”，避免虚拟摇杆遮挡操作。

### 5.2 判定系统

- 玩家飞机视觉尺寸可以较大，但受击判定点应较小。
- 敌弹、导弹、拾取物必须有清晰颜色区分。
- 玩家受击后有 1-2 秒无敌闪烁。

### 5.3 武器系统

Demo 首版武器：

- 机炮：默认直线射击。
- 双联机炮：拾取升级后左右双线。
- 机载追踪导弹：通过导弹补给道具获得，自动锁定前方或最近的高威胁目标。

后续武器：

- 激光脉冲
- 扇形弹幕
- 贯穿弹
- 空地炸弹
- EMP 清弹

### 5.4 导弹系统

导弹系统分为敌方地对空导弹和玩家机载导弹，两者使用不同颜色、音效和行为规则。

敌方地对空导弹：

- 来源：导弹车、固定式 SAM 发射架、Boss 防空车。
- 流程：雷达锁定提示 -> 发射烟迹 -> 导弹进入追踪 -> 燃料耗尽或飞出屏幕后失效。
- 锁定提示：玩家附近出现红色警示框或屏幕边缘方向提示，同时播放导弹锁定音。
- 追踪规则：导弹持续朝玩家当前位置修正方向，但必须有最大转向角和最大追踪时间。
- 躲避规则：玩家通过横向变向、低速精密移动或 EMP 清弹躲避。
- 平衡要求：导弹不能瞬间贴脸生成；发射前至少给 0.6-1.0 秒预警。

玩家机载导弹：

- 来源：拾取“导弹补给”道具后获得。
- 持续方式：建议 Demo 采用弹药制，例如获得 6-10 发导弹，而不是永久解锁。
- 发射方式：玩家继续自动射击，导弹按冷却间隔自动发射。
- 锁定优先级：Boss 部件 > 导弹车/SAM > 轻型战机 > 无人机。
- 伤害定位：单发高伤害、射速低，适合清除高威胁目标。
- 视觉区分：玩家导弹使用蓝白尾焰，敌方导弹使用红橙尾焰。
- 平衡要求：导弹补给不应频繁掉落，避免玩家长期无脑清屏。

### 5.5 拾取系统

Demo 首版掉落：

- 火力升级
- 导弹补给
- 维修包
- 护盾
- 分数倍率

掉落节奏应服务关卡体验，不应让玩家持续满屏无脑输出。

### 5.6 评分系统

关卡结束后显示：

- 击毁率
- 连击最高数
- 受击次数
- 剩余生命
- 通关用时
- 综合评分：C / B / A / S

## 6. 美术方向

### 6.1 总体风格

采用 2D 俯视街机风格。视觉重点是“现代军事轮廓 + 高可读弹幕 + 沙漠/山地/城市环境”。不追求照片写实，避免过暗、过灰、过脏导致敌弹和飞机难以辨认。

### 6.2 玩家飞机

Demo 玩家机 AFX-35：

- 轮廓可参考现代隐身战斗机：三角翼、双垂尾、锋利机头。
- 必须是原创变体，不直接复刻真实照片或厂商图片。
- 不放真实国旗、军徽和编号。
- 可以用蓝白冷色灯光表现玩家阵营。

### 6.3 敌方单位

敌方视觉可使用沙色、暗绿、红色警示灯作为识别色。  
无人机、导弹车、防空炮应夸张化，让玩家一眼理解威胁类型。

### 6.4 伊朗风格环境参考

可以参考以下视觉元素：

- 沙漠公路
- 山地峡谷
- 土黄色城市建筑
- 拱形窗、穹顶、宣礼塔轮廓
- 工业设施、雷达站、地下掩体入口

注意：

- 民用建筑主要作为背景，不作为鼓励攻击的目标。
- 不描绘平民伤亡。
- 不使用真实宗教符号作为攻击目标。

## 7. 音效与音乐方向

### 7.1 音乐

Demo 第一关音乐应紧张、快速、现代电子军鼓风格。  
节奏参考街机射击，不做过度写实战争氛围。

音乐层次：

- 关卡普通段：120-140 BPM。
- Boss 段：增加低频、打击乐和警报纹理。
- 胜利结算：短促、有成就感。

### 7.2 音效

必需音效：

- 玩家机炮
- 敌方机炮
- 导弹发射
- 导弹锁定警告
- 爆炸，小/中/大三档
- 拾取道具
- 玩家受击
- 护盾触发
- Boss 阶段切换
- UI 点击和结算

音效注意：

- 自动射击音效不能刺耳，需短促、低疲劳。
- 导弹警告音要明显，但不能过频繁。
- 多个爆炸同时播放时需要限制音量叠加。

## 8. 技术概要

推荐技术栈：

- TypeScript
- Phaser 3
- Vite
- WebAudio
- localStorage

建议目录：

```text
src/
  main.ts
  scenes/
    BootScene.ts
    MenuScene.ts
    GameScene.ts
    ResultScene.ts
  entities/

  data/
    aircraft.ts
    enemies.ts
    levels.ts
    weapons.ts
assets/
  sprites/
  audio/
  music/
  ui/
```

### 8.1 Web 性能目标

- 桌面浏览器：稳定 60 FPS。
- 中端手机浏览器：稳定 45-60 FPS。
- 同屏弹体数量 Demo 阶段建议控制在 120 个以内。
- 所有子弹、敌人、爆炸特效使用对象池。
- 背景使用循环贴图或分段图片，避免超大图直接滚动。

### 8.2 Android 扩展

后续使用 Capacitor 或同类方案打包为 Android。  
从 Demo 阶段起应注意：

- 竖屏优先。
- UI 避开刘海屏和系统手势区。
- 所有核心操作支持触摸。
- 音频需要处理切后台暂停。
- 存档接口封装，便于后续接 Android 本地存储或云存档。

## 9. Demo 验收标准

Demo 完成时应具备：

- 可在浏览器打开并进入游戏。
- 支持键盘和触摸移动。
- 玩家飞机自动射击。
- 至少 3 种敌方单位。
- 至少 2 种地面单位。
- 至少 2 种拾取道具。
- 完整第一关“闪电攻击”。
- 一个小 Boss。
- 游戏开始、暂停、失败、胜利、结算界面。
- 本地最高分记录。
- 基础音效和一首循环音乐。

## 10. 内容合规与表达注意事项

- 不使用真实人物作为打击目标。
- 不呈现现实暗杀流程。
- 不给出现实武器操作、攻击路线、基地坐标、目标坐标等可操作信息。
- 不使用真实国家旗帜、军徽、部队番号作为可攻击对象。
- 多阵营内容应体现玩法差异，不以现实民族、宗教或国家仇恨为卖点。
- 可以使用“公开装备外形启发”，但美术资产必须原创。
